## 项目编码规则

### 格式
`2025H1-[阶段代码]-[序号]`

### 阶段代码
- **SG**：施工图 (Construction Drawing)
- **FA**：方案 (Scheme)
- **SC**：施工配合 (Site Cooperation)

### 示例
- `2025H1-SG-001`：2025年上半年施工图阶段第1个项目
- `2025H1-FA-001`：2025年上半年方案阶段第1个项目

## 导入说明

### 1. 使用导入脚本
```bash
# 执行导入脚本
node import-projects-2025-h1.js
```

### 2. 导入内容
- 项目编码（code）
- 项目名称（name）
- 项目阶段（stage）
  - `construction`：施工图
  - `scheme`：方案
- 项目状态（status）：`active`

### 3. 待补充信息
导入后需要在系统中补充以下信息：
- 建筑面积（building_area）
- 设计费单价（design_fee_unit_price）
- 空调面积表
- 参与人员分配

## 注意事项

1. **项目编码唯一性**
   - 每个项目都有唯一的编码
   - 重复导入时会自动跳过已存在的项目

2. **阶段映射**
   - Excel中的"施工图" → 系统中的 `construction`
   - Excel中的"方案" → 系统中的 `scheme`
   - Excel中的"施工配合" → 系统中的 `construction`

3. **数据完整性**
   - 初始导入时，建筑面积和设计费单价为0
   - 需要后续在系统中手动补充
   - 补充完整后才能计算提成

4. **重复项目处理**
   - "花都国际绿色建材中心"同时出现在施工图和施工配合中
   - 使用不同的项目编码区分（SG-001 和 SC-001）

## 后续操作

### 1. 补充项目信息
登录系统后，为每个项目补充：

- 设计费单价
- 其他必要信息

### 2. 配置空调面积表
对于需要计算空调提成的项目：

- 配置空调面积表
- 系统自动计算部门分配

### 3. 配置发放节点
为项目配置提成发放计划：
- 设置发放日期
- 设置发放阶段
- 设置发放比例

### 4. 分配参与人员
在第三级进行个人分配：
- 选择参与人员
- 分配提成金额
- 关联发放节点
